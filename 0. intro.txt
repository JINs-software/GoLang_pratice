[golang 코드가 실행되기까지]
1. 폴더 생성
    - go 언어에서 모든 코드는 '패키지' 단위로 작성됨
    - 같은 폴더에 위치한 .go 파일은 모두 같은 패키지에 포함되고, 패키지명으로 폴더명을 사용
    - ex) goproject/hello/extra
        - hello 폴더 내 .go 파일은 모두 hello 패키지에 포함됨
        - extra 폴더 내 .go 파일은 모두 extra 패키지에 포함됨

2. .go 파일 생성 및 작성

3. go 모듈 생성
    - go 1.16 버전 이후로 go 모듈이 기본적으로 적용됨
    - 모든 go 코드는 빌드 전 모듈을 생성해야 함
    - 모듈 생성: go mod init [모듈 이름]
    - 모듈 생성 후 go.mod 파일 생성됨
        - 모듈명 / go 버전 / 필요 패키지 목록 정보 등이 담김

4. 빌드
    - go build 명령으로 go 코드를 기계어로 컴파일하여 실행 파일로 만듬
    - GOOS와 GOARCH 환경변수를 조정해서 다른 운영체제와 아키텍처에서 실행되는 실행 파일을 만들 수 있음

5. 실행


[package main]
- 패키지 선언은 해당 go 코드가 어느 패키지에 속하는지 알려줌
- 패키지는 코드 묶음이고, 여러 기능을 제공함
- package main은 "프로그램 시작점을 포함하는 특별한 패키지"
- 즉, main() 함수가 없는 패키지는 패키지 이름으로 main을 쓸 수 없음
